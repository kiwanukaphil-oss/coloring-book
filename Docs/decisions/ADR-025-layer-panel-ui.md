# ADR-025: Layer Panel UI

## Status
Accepted

## Context
Phase 3 adds a user-visible layer panel so users can manage their drawing layers. The panel
must integrate with both UI modes (ADR-015), follow accessibility patterns (ADR-013), and
use existing CSS design tokens (ADR-010).

## Decision

### Panel placement and visibility
- **Studio mode**: panel is visible on the right edge of the viewport, vertically centered
- **Kids mode**: panel is hidden via `.studio-only` CSS class (ADR-015); Kids mode draws on layer 0 only

### DOM structure
```html
<div id="layer-panel" class="layer-panel studio-only" role="region" aria-label="Layers panel">
  <div class="layer-panel-header">
    <span class="layer-panel-title">Layers</span>
    <button id="add-layer-btn" class="layer-add-btn" aria-label="Add layer">+</button>
  </div>
  <ul id="layer-list" class="layer-list" role="listbox" aria-label="Layers"></ul>
</div>
```

Each layer item is generated by `LayerPanel.renderLayerItem()` and appended to `#layer-list`:
```html
<li class="layer-list-item [active]" role="option" aria-selected="[true|false]"
    tabindex="0" data-layer-index="N">
  <canvas class="layer-thumbnail" width="40" height="40"></canvas>
  <span class="layer-name">Layer N</span>
  <button class="layer-visibility-btn" aria-label="[Show|Hide] layer N">👁</button>
  <input type="range" class="layer-opacity-slider" min="0" max="100" value="100"
         aria-label="Layer N opacity">
  <button class="layer-delete-btn" aria-label="Delete layer N">✕</button>
</li>
```

### Interactions
| Interaction | Action |
|-------------|--------|
| Click layer item | `LayerManager.setActiveLayer(index)` |
| Click visibility button | `LayerManager.setLayerVisibility(index, !current)` |
| Drag opacity slider | `LayerManager.setLayerOpacity(index, value / 100)` |
| Click Add button | `LayerManager.addLayer()` |
| Click Delete button | snapshot → `CommandManager.createLayerDeleteCommand` → `LayerManager.deleteLayer(index)` |
| Drag layer item | `LayerManager.reorderLayer(fromIndex, toIndex)` |

Add button is `disabled` when `LayerManager.getLayerCount() >= 5`.
Delete button is visually hidden (`.visually-hidden` class, ADR-003) when `getLayerCount() <= 1`
to preserve layout stability.

### Drag-to-reorder

Layer items are `draggable="true"`. HTML5 drag events on each `<li>` implement reordering:

- **`dragstart`**: stores the item's logical `layerIndex` via
  `e.dataTransfer.setData('text/plain', String(index))`
- **`dragover`**: calls `e.preventDefault()` (required to allow drop) and adds `.drag-over`
  CSS class to the drop-target item for visual feedback (ADR-003)
- **`dragleave`**: removes `.drag-over` class
- **`drop`**: reads `fromIndex` from `dataTransfer`, calls
  `LayerManager.reorderLayer(fromIndex, toIndex)`, removes `.drag-over`

**Index mapping note:** `LayerPanel.refresh()` renders the list in reverse visual order
(top layer first). The `index` stored in `dataTransfer` is the logical LayerManager index,
not the visual position. `reorderLayer(fromIndex, toIndex)` receives logical indices, so the
mapping is always correct regardless of visual order.

### Keyboard accessibility (ADR-013)
- Layer items are `tabindex="0"` — reachable by Tab
- `Enter` or `Space` on a layer item activates it
- `Space` on the visibility button toggles visibility
- Focus indicator: `outline: 3px solid var(--color-focus)` (3px per ADR-013)
- `role="listbox"` + `role="option"` with `aria-selected` matches ARIA listbox pattern

### Thumbnail generation
Each layer item contains a `<canvas class="layer-thumbnail">` that is drawn by
`LayerPanel.refresh()` by scaling the layer's canvas onto the 40×40 thumbnail canvas.
The thumbnail is regenerated on every `refresh()` call.

### EventBus integration
`LayerPanel` listens on EventBus for:
- `layer:active-changed` → `refresh()`
- `layer:added` → `refresh()`
- `layer:deleted` → `refresh()`
- `layer:visibility-changed` → `refresh()`
- `layer:opacity-changed` → `refresh()`

`LayerManager` emits these events when state changes.

### Rules
- The layer panel is shown only in Studio mode — hidden in Kids mode via `.studio-only` class (ADR-015)
- Layer panel uses only CSS design tokens (ADR-010); no hardcoded colour or size values
- All interactive elements have visible focus indicators ≥ 3px (ADR-013)
- Delete button hidden (not removed) when only 1 layer exists, to preserve layout
- Add button disabled (not hidden) when 5 layers exist
- Thumbnails refresh on every `LayerPanel.refresh()` call
- Layer names are fixed strings "Layer 1", "Layer 2", etc. (not editable)
- Delete is undoable via CommandManager (ADR-026); add and reorder are not undoable
- Drag-to-reorder uses logical layer indices, not visual positions
